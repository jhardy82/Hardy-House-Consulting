Which 3D representation for Metatron's Cube?

All three build on existing code — the difference is geometric depth and wow-factor.

A — Vector Equilibrium (Recommended)

13 glowing spheres in 3D space — 1 centre + 12 cuboctahedron vertices — connected by all 78 Metatron lines. Fully orbitable. Uses existing OrbitControl + buildGlowEdge patterns.

★ Most accurate sacred geometry · reuses existing code · cleanest visual

B — 64-Tetrahedron Grid

The "isotropic vector matrix" — 64 tetrahedra tiling 3D space. Spectacular but complex. No direct mapping to existing CLAUDE.md invariants. Requires new geometry module.

More impressive at first glance · harder to implement correctly · less sacred-geometry-accurate for Metatron specifically

C — Animated 2D→3D Morph

Start from the existing flat MetatronScene, then animate the 13 centres rising into 3D space. A unique "construction" reveal. More animation engineering than geometry.

Visually dramatic · complex animation state · diverges from existing scene pattern

Context

The existing #geometry section already has: (1) a 2D MetatronScene with 78 flat lines, (2) a Cuboctahedron shape with outer-shell edges only. Option A fills the gap — 13 spheres in true 3D with all 78 internal connecting lines, fully orbitable. It directly extends CLAUDE.md invariants to 3D.
